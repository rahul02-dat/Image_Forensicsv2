# Image Forensics v2

This project provides a robust solution to distinguish real photographs from AI-generated images (created via GANs, Stable Diffusion, Midjourney, etc.). It is trained on the [GenImage](https://github.com/GenImage-Dataset/GenImage) dataset and specifically designed to be resilient against social media compression.

## About the Algorithm

The core engine is a Dual-Stream Convolutional Neural Network designed to detect AI-generated imagery even after it has been degraded by social media platforms:

- **Spatial Stream:** An EfficientNet-B4 backbone that captures visual artifacts, textures, and anomalies in the pixel domain.
- **Frequency Stream:** A custom Patch-DCT (Discrete Cosine Transform) module. In v2, it strategically targets the mid-frequency coefficients, skipping the low-frequency band which is heavily corrupted by JPEG compression (like on WhatsApp or Instagram) while retaining enough information to identify generative fingerprints.
- **Cross-Modal Attention Fusion:** A custom attention mechanism that optimally fuses the representations from the spatial and frequency streams before the final classification step.
- **Social Media Preprocessing:** A preprocessing step that estimates JPEG compression quality and applies adaptive unsharp masking to partially recover high-frequency details.
- **Test-Time Augmentation (TTA):** The prediction endpoint averages probabilities across multiple augmented crops and flips of the image, further boosting inference robustness.

## About GradCAM

To provide explainability, the detector employs Gradient-weighted Class Activation Mapping (GradCAM):
- It performs a single forward pass with gradients enabled on the preprocessed image.
- It generates a heatmap overlay highlighting the specific spatial regions that most strongly triggered the model's decision.
- This allows users to understand *why* the model predicted an image as REAL or AI-GENERATED by looking at the specific artifacts or textures the network focused on.

## Test Results

The model was evaluated on a held-out test set from the GenImage dataset (2,400 images: 1,184 Real, 1,216 AI-generated). With Test-Time Augmentation (TTA) enabled, the model achieved the following performance metrics:

- **Accuracy:** `84.50%`
- **ROC-AUC:** `0.9209`
- **F1-Score:** `0.8463`

*Note: The model is highly robust to double-compression and heavy resizing typical of social media sharing, which was heavily simulated during training.*

## Proper Instructions

### Prerequisites
- Python 3.12+
- `pip`

### 1. Model Setup
1. Train the model using the provided Jupyter notebook pipeline (`image.ipynb`).
2. Retrieve the resulting best checkpoint model.
3. Place the file precisely at `checkpoints/best_model.pt` in this repository.

### 2. Local Development Server
To run the FastAPI application locally:
```bash
pip install -r requirements.txt
uvicorn app:app --host 0.0.0.0 --port 8000 --reload
```
The API and UI will be available at `http://localhost:8000`.

### 3. Docker Deployment
This project includes a Dockerfile configured to run in containerized environments (like Hugging Face Spaces) using Port `7860`.
```bash
docker build -t forensic-app .
docker run -p 7860:7860 forensic-app
```
*(If deploying on Hugging Face Spaces, simply uploading the checkpoint to `checkpoints/best_model.pt` in the Space's file browser will automatically reload the application).*
